# 🤖 Questerix Content Engine: The Curriculum Power-Tool

This repository contains the "Intelligence Layer" of the Questerix platform. It is designed to transform raw educational content into structured, high-quality curriculum data.

## 🏗️ Architecture at a Glance

The engine is split into two distinct parts:

1.  **The Python Engine (`/src`)**: A heavy-duty CLI tool for document parsing (PDF, DOCX, Images) and bulk question generation.
2.  **The Cloudflare Workers (`/workers`)**: The "Live API" that connects to your Admin Panel for real-time AI features and email alerts.

---

## 🚀 Part 1: Using the Python Engine (Local Power-Tool)

Use this when you have large documents and want to generate hundreds of questions at once.

### 1. Setup

```bash
# Initialize the workshop
python -m venv .venv
source .venv/bin/activate # Windows: .venv\Scripts\activate
pip install -r requirements.txt
```

### 2. The Command Matrix

| Action       | Command                            | Description                     |
| :----------- | :--------------------------------- | :------------------------------ |
| **Extract**  | `python -m src extract lesson.pdf` | Pulls raw text from a document. |
| **Generate** | `python -m src generate text.txt`  | Turns text into JSON questions. |
| **Pipeline** | `python -m src pipeline intro.pdf` | The "Auto-Pilot": PDF → JSON.   |

### 3. Workflow Recommendation

1. Put your PDF in the root.
2. Run the `pipeline` command with a target `skill-id`.
3. Take the resulting `questions.json` and upload it to the **Admin Panel -> Bulk Import**.

---

## ⚡ Part 2: Using & Deploying Workers (The App Bridge)

These workers are the "Live" bits that your Admin Panel calls every day.

### 1. How were they used?

- **AI Sync**: The `parse-import-prompt` worker handles unstructured text in the Bulk Import page.
- **Alerts**: The `email` worker handles security alerts and system notifications.

### 2. Deployment

```bash
cd workers
npm install
npx wrangler deploy
```

## 📜 Curriculum Standards (Data Schema)

Questions generated by this engine follow a strict JSON schema required by the Questerix Admin Panel:

```json
{
  "content": "What is 2 + 2?",
  "type": "multiple_choice",
  "options": {
    "options": [
      { "id": "a", "text": "3" },
      { "id": "b", "text": "4" }
    ]
  },
  "solution": { "correct_option_id": "b" },
  "points": 1
}
```

## 🛠️ Maintenance & Hygiene

This repo is equipped with high-standard engineering tools:

- **Secret Protection**: Gitleaks is configured to prevent API key leaks.
- **Spell Check**: CSpell keeps the educational content professional.
- **Testing**: Run `pytest` for Python or `npm test` inside `/workers`.
